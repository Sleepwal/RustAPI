### 🛡️ 错误处理改进

#### 自定义错误类型 (ApiError)

**问题**：之前使用 `String` 传递错误，缺乏结构化处理，错误消息不够友好。

**优化方案**：
- 添加 `ApiError` 枚举类型：
  - `InvalidUrl(String)` - URL 格式错误
  - `NetworkError(String)` - 网络请求失败（连接错误、超时等）
  - `ResponseReadError(String)` - 响应体读取失败
- 实现 `user_message()` 方法，提供用户友好的错误消息
- 实现 `Display` trait，支持格式化输出

**收益**：
- 错误类型明确，便于代码审查和维护
- 错误消息更清晰，提升用户体验
- 为未来的错误分类和处理奠定基础

---

### ⏱️ 请求超时配置

#### 可配置的请求超时时间

**问题**：之前没有超时限制，如果服务器无响应，应用会无限等待。

**优化方案**：
- 在 `ApiRequest` 中添加 `timeout_secs` 字段（默认 30 秒）
- 在 `http::send_request` 中使用 `tokio::time::timeout` 包装请求
- UI 中添加 DragValue 控件，支持 1-300 秒范围配置
- 超时触发友好的错误消息："Request timed out after Xs"

**收益**：
- 防止应用因无响应服务器而卡死
- 用户可根据网络环境和服务器性能自定义超时时间
- 超时后快速失败，节省用户时间

---

### ⌨️ 键盘快捷键

#### 提升操作效率

**新增快捷键**：
- **Ctrl+Enter** 或 **Ctrl+R**: 发送请求
  - 请求进行中时自动忽略，防止重复发送
- **Ctrl+L**: 清空所有历史记录

**收益**：
- 减少鼠标点击，提升操作效率
- 符合常见 IDE 和工具的快捷键习惯
- 特别适合频繁测试 API 的开发者

## 2026-04-11 - 性能优化与代码质量提升

这次更新专注于性能和代码质量，让应用更快、更稳定！🚀

---

### ⚡ 性能优化

#### HTTP 客户端复用

**问题**：之前每次发送请求都会创建新的 `reqwest::Client` 实例，导致：
- 连接池每次都重建
- 资源浪费严重

**优化方案**：
- 在 `ApiClientApp` 结构体中持有 `reqwest::Client` 实例
- 修改 `send_request` 函数签名，接收 `&Client` 参数
- 异步任务通过 Arc 包装的 Client 引用发送请求

**收益**：
- 避免重复的连接池创建开销
- 请求发送速度更快
- 减少资源分配和释放

#### Arc 共享响应数据

**问题**：响应数据在历史记录保存和 UI 渲染时会被多次 `clone`，对于大型响应体（如几 MB 的 JSON）会造成大量内存分配和拷贝。

**优化方案**：
- `ApiResponse.headers` 改为 `Arc<Vec<(String, String)>>`
- `HistoryItem.response` 改为 `Option<Arc<ApiResponse>>`
- UI 渲染时直接引用，避免不必要的 clone

**收益**：
- 显著降低内存分配和拷贝开销
- 历史记录操作更快
- 响应面板渲染更流畅

#### 历史记录数量限制

**问题**：历史记录可以无限增长，长时间使用后可能导致内存占用过高。

**优化方案**：
- 添加 `MAX_HISTORY_SIZE = 100` 常量限制
- 实现 FIFO 策略，超限时自动移除最旧记录
- 为 `RequestHistory` 添加 `add()`、`clear()`、`len()`、`is_empty()` 方法

**收益**：
- 防止内存泄漏
- 保持应用响应性
- 自动管理，无需用户手动清理

### 🛡️ 健壮性提升

#### URL 验证与自动补全

**新增功能**：
- 添加 `ApiRequest::validate_and_normalize_url()` 方法

- 自动补全缺失的 `https://` 协议前缀（如输入 `example.com` 自动转为 `https://example.com`）

**用户体验**：
- 减少因 URL 格式错误导致的请求失败
- 输入更友好，不用每次都手动输入协议前缀
